# Rug Pull Detector

A machine learning system that detects DeFi rug pulls on Uniswap V2. Given a token pair, it predicts the probability that the creator will drain liquidity and leave investors with worthless tokens.

---

## What Is a Rug Pull?

A rug pull is a crypto scam where a token creator adds liquidity to a DEX pool, attracts investors, then removes all the liquidity — crashing the price to zero. This project uses on-chain behavioral patterns to flag these tokens before they collapse.

---

## Dataset

- **File:** `Dataset_v1.9.csv`
- **Size:** 18,296 Uniswap V2 token pairs
- **Class split:** 90% rug pulls (16,462) / 10% normal (1,834)
- **Sources:** TheGraph (Uniswap V2 subgraph) + Ethplorer token holder API

---

## Features

### 18 Base Features (from on-chain data)

**LP / Transaction Activity**
| Feature | Description |
|---|---|
| `mint_count_per_week` | Liquidity add events per week |
| `burn_count_per_week` | Liquidity remove events per week |
| `mint_ratio` | Mints / total events |
| `swap_ratio` | Swaps / total events |
| `burn_ratio` | Burns / total events |
| `mint_mean_period` | Average timing of mints across token lifetime (0=start, 1=end) |
| `swap_mean_period` | Average timing of swaps across token lifetime |
| `burn_mean_period` | Average timing of burns across token lifetime |
| `swap_in_per_week` | Buy-side swaps (ETH → token) per week |
| `swap_out_per_week` | Sell-side swaps (token → ETH) per week |
| `swap_rate` | swap_in / (swap_out + 1) — buy pressure ratio |

**LP Holder Features**
| Feature | Description |
|---|---|
| `lp_avg` | Average LP token share among significant holders |
| `lp_std` | Standard deviation of LP token distribution |
| `lp_creator_holding_ratio` | % of LP tokens held by the token creator |
| `lp_lock_ratio` | % of LP tokens in a known locker contract |

**Token Holder Features**
| Feature | Description |
|---|---|
| `token_burn_ratio` | % of token supply sent to burn addresses |
| `token_creator_holding_ratio` | % of token supply held by the creator |
| `number_of_token_creation_of_creator` | Number of ERC-20 tokens this creator has previously deployed |

### 8 Derived / Engineered Features (computed in `train.py`)

| Feature | Description |
|---|---|
| `unlocked_creator_lp` | Creator LP share × (1 − lock ratio) — direct rug pull capability |
| `burn_to_mint_ratio` | LP burns/week ÷ mints/week — high = liquidity draining faster than added |
| `total_swap_volume` | Buy + sell swaps per week — very low = abandoned token |
| `creator_total_exposure` | Token creator % + LP creator % combined — single actor control |
| `sell_dominance` | Sell swaps / total swaps — near 1.0 = everyone is trying to exit |
| `swap_timing_delay` | Average swap timing minus average mint timing — large gap = pump and dump |
| `lp_exposure_risk` | Binary: 1 if creator holds >50% of unlocked LP |
| `honeypot_signal` | Buy swaps / sell swaps — very high = sells are blocked |

## Models

Three models are trained and saved to `models/`:

| Model | File |
|---|---|
| Logistic Regression | `models/logistic_regression.pkl` |
| Random Forest | `models/random_forest.pkl` |
| XGBoost (tuned) | `models/xgboost_best.pkl` |
| MinMaxScaler | `models/scaler.pkl` |
| Feature names | `models/feature_names.pkl` |

### Results (Test Set — 3,660 tokens)

| Model | Accuracy | Precision | Recall | F1 | AUC-ROC |
|---|---|---|---|---|---|
| Logistic Regression | 92.16% | 98.74% | 92.47% | 95.50% | 97.18% |
| Random Forest | 97.32% | 99.20% | 97.81% | 98.50% | 99.06% |
| **XGBoost (tuned)** | **97.62%** | **99.08%** | **98.27%** | **98.67%** | 98.83% |

**XGBoost is the primary model.** It missed only 57 actual rug pulls out of 3,293 in the test set.

### Training Details
- 80/20 stratified train/test split (preserves the natural 90/10 class ratio in both halves)
- MinMaxScaler normalization
- SMOTE oversampling applied to training set only (balances the minority normal class for learning)
- XGBoost tuned via 5-fold GridSearchCV optimizing F1 score

## How to Run

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Train all models
```bash
python train.py
```
Saves all model files to `models/`.

### 3. Generate evaluation plots
```bash
python evaluate.py
```
Saves 7 plots to `plots/`:
- `confusion_matrices.png`
- `roc_curves.png`
- `precision_recall_curves.png`
- `feature_importance_xgb.png`
- `feature_importance_rf.png`
- `correlation_matrix.png`
- `feature_distributions.png`

### 4. Score a live token
```python
import pickle
import pandas as pd
from features import extract_features
from train import add_derived_features

with open('models/xgboost_best.pkl', 'rb') as f: model = pickle.load(f)
with open('models/scaler.pkl', 'rb') as f: scaler = pickle.load(f)
with open('models/feature_names.pkl', 'rb') as f: feature_names = pickle.load(f)

raw = extract_features(
    pair_id='0xYOUR_PAIR_ADDRESS',
    token_id='0xYOUR_TOKEN_ADDRESS',
    eth_index=1   # 0 if token0 is WETH, 1 if token1 is WETH
)

df = add_derived_features(pd.DataFrame([raw]))
X = scaler.transform(df[feature_names].values)
prob = model.predict_proba(X)[0][1]
print(f"Rug pull probability: {prob:.1%}")
```

## Known Limitations / Next Steps

- `number_of_token_creation_of_creator` is currently hardcoded to `1` — needs a real Ethplorer/Etherscan lookup
- LP lock expiry check is not implemented — requires Bitquery API
- TheGraph hosted service is deprecated — needs migration to the decentralized network with an API key
- Dataset and feature extraction are Uniswap V2 only — no V3 support yet
- No real-time pipeline or alerting system built yet
